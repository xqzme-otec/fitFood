# Report on LLM Usage – FitFood

This report describes how AI/LLM tools were used throughout the Assignment 5 sprint. It covers all AI/LLM-assisted activities, including coding, project management, workflow enforcement, architecture documentation, and quality assurance, explicitly stating which parts of the work were done without any AI assistance.

## Summary
AI/LLM tools were heavily utilized during this Sprint for development support, complex documentation setup, and Agile project management. As the Sprint focused on delivering MVP v2, formalizing Architecture, and responding to customer feedback, LLM usage shifted towards extensive text analysis, prompt engineering for the product's core feature, and structuring Architecture Decision Records (ADRs).

The division of labor for this sprint was:
* **DeepSeek:** Used by the development team to assist with recipe matching logic, prompt engineering for the LLM-based meal plan generation feature, and KBZHU fallback calculations.
* **Gemini:** Used extensively by the Scrum Master for Agile workflow management, analyzing a massive multi-hour customer UAT transcript for exact feedback extraction, diagnosing team/process friction, and formatting architecture documentation.
* **Claude (Claude Code):** Used to realize the MVP v2 frontend/backend integrations (e.g., receipt scanner API connection) and assist with Docker configuration troubleshooting.
* **Manual Work:** UAT execution, peer reviews, core architectural decision-making, and deployment verification were done strictly by the team.

## How Each Tool Was Used

### Gemini — Agile Management & Analytical Support
Gemini was the primary tool used for Project Management and Assignment 5 compliance:
* **Transcript & Feedback Analysis:** Deeply analyzing the raw transcripts from the Customer UAT session (which included extensive deployment troubleshooting) to identify communication gaps, isolate exact customer requirements (e.g., Chestny Znak integration, fridge-to-meal deduction), and translate them into actionable Product Backlog Items.
* **Error Diagnostics:** Analyzing team workflow friction and errors in communication to formulate a highly accurate and objective Sprint Retrospective and Reflection.
* **Documentation & Syntax Correction:** Structuring the new Architecture documentation (static, dynamic, and deployment views) and formatting the Architecture Decision Records (ADRs) to ensure compliance with strict course artifact semantics.

### Claude — Implementation / Realization
Claude (via Claude Code) was used to accelerate the technical requirements of Assignment 5:
* Implementing the MVP v2 features, including the receipt scanner file upload integration and the UI for the new "Generate Meal Plan" flow.
* Assisting with Dockerfile analysis to address the customer's feedback regarding image bloat (targeting a reduction to ~200 MB).

### DeepSeek — Logic and Algorithm Design
DeepSeek was used by the developers to support technical tasks:
* Refining the tokenization and semantic chunking logic to accurately match ingredients from the user's virtual fridge with the recipe database.
* Finalizing the prompt structures for the LLM-based meal plan generation engine and the KBZHU fallback estimation for unmatched products.

## What Was NOT AI-Assisted
* **Live Customer UAT:** The actual User Acceptance Testing session, local deployment troubleshooting, and live feedback collection were conducted entirely by human team members.
* **Final Code Reviews & Merges:** All GitHub Pull Requests, code reviews, and manual approvals (enforcing branch protection rules) were performed strictly by team members.
* **Architectural Decision Making:** The actual technical choices recorded in the ADRs (e.g., how to handle ingredient variants, choosing the RAG-adjacent approach) and final project priorities were determined by the team, not generated by AI.
* **Infrastructure Deployment:** Testing the VM access and running the Docker containers locally during the presentation was done manually.
