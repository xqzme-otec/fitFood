# Sprint 5 (Week 7) Reflection

This final sprint taught us important lessons about handing over a software project. Writing code is only part of the work; making sure it runs properly and is documented clearly is just as important.

**Key Learnings from Final Transition:**
* **Deployment as a Blocker:** We learned that deployment issues can become a major bottleneck. Our main challenge was fixing the deployment on the university network so that the customer could easily set it up and run it themselves. We realized that these technical setup issues must be resolved as early as possible to avoid stress at the end of the project.
* **The Importance of Documentation:** Writing the `customer-handover.md` document showed us how much implicit knowledge we had. Explaining our backend and `.env` setup in simple, step-by-step instructions proved that clear documentation is just as important as the code itself.
* **Final Delivery and Code Freeze:** Releasing MVP v3 taught us to focus on finishing the product. We had to stop adding new features and instead focus entirely on fixing bugs, ensuring stability, and preparing the app for the final handover.
