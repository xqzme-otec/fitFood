# Team Process Requirements

## 1. Branching Strategy
* All new features and bug fixes must be developed in separate feature branches.
* Branch naming convention: `<issue-number>-short-description` (e.g., `21-inventory-add`).
* The `main` branch is protected and contains the stable MVP release.

## 2. Code Review & Merging
* Direct commits to the `main` branch are restricted.
* All changes must be integrated via Pull Requests (PRs).
* At least one peer review and approval is required before a PR can be merged.
